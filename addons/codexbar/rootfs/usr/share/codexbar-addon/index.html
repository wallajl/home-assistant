Quick path

  1. Enable the providers you care about below.
  2. Paste API keys/cookies only where needed. Leave local CLI providers blank if you will copy auth files into the add-on config folder.
  3. Press Save setup, then Test a provider.

Config is written to /config/codexbar/config.json with 0600 permissions. Secrets stay inside this add-on container/config storage.

OAuth login

Codex and Claude subscription limits use OAuth, not normal API keys.

This add-on cannot safely receive your provider password. Log in with the normal Codex/Claude CLI on any computer, then upload the generated OAuth JSON below. The add-on saves it with 0600 permissions.

Providers

Local CLI auth files

Codex and Claude often need local CLI credential files instead of API keys. Copy them into the add-on config storage, then restart this add-on.

Codex files:  /config/.codex
Claude files: /config/.claude
CodexBar config: /config/codexbar/config.json
